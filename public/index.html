This website is designed to generate pictorial representations of the pipelining of instructions through a CPU, like the prototype example above.

The website only suppports the following, case sensitive, instructions:

"ldur","stur","mov","movz","movk" "add","sub","addi","subi","and","ior","eor"

Which are defined in the lecture content of the Computer Systems Architecture unit. (Note that these instructions differ slightly from the true ARMv8 instruction set).

The other key limitation of this website currently is that it can only be used with 5 instructions, no more, no less. Examples of assembly instructions which can be uploaded to the website can be found here. See the readme for the formatting rules that must be adhered to when creating your own sequences of instructions.

There are 2 ways of creating the pipeline. A manual version where the user chooses the action of each instruction, and an automatic version that generates the pipeline for the user. There is a toggle which turns data forwarding from the ALU output on or off, so that the difference can be observed.

It is recommended for students to try creating the pipeline manually first and then compare the results with the generated pipeline as a learning exercise.
PLEASE NOTE: Use of the manual controls will invalidate the automatically generated pipeline, even if all pipeline stages are removed using the undo button. The page MUST be refreshed before the pipeline can be automatically generated.

For curious students, the source code for this website can be viewed here. This website has not been tested very extensively, and therefore it is possible there are bugs in the pipeline generation. If you believe to have found a bug please report it as an issue on the github and I will attempt to resolve it.
