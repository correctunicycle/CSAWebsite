This website is designed to generate pictorial representations of the pipelining of instructions through a CPU, like the prototype example above.

To use this tool first you must download the ARM GNU toolchain, which can be found here. Once downloaded, you must add the /bin folder to your path. you can then assemble assembly language using the following command:

arm-none-eabi-as Hello.S

This will create an output file called a.out For this to be readable to the human eye and for the webpage this must be coverted into plain text, which can be done using the objdump command

arm-none-eabi-objdump -D -j .text a.out > yourOutputFile.txt

You can then upload this file to the website and view the pipeline animation, although please note that the website only suppports the following, case sensitive, instructions:

"ldur","stur","mov","movz","movk" "add","sub","addi","subi","and","ior","eor"

There are 2 ways of creating the pipeline. A manual version where the user chooses the action of each instruction, and an automatic version that generates the pipeline for the user.

It is recommended for students to try creating the pipeline manually first and then compare the results with the generated pipeline as a learning exercise.
PLEASE NOTE: Use of the manual controls will invalidate the automatically generated pipeline, even if all pipeline stages are removed using the undo button. The page MUST be refreshed before the pipeline can be automatically generated.
